# AI 视频总结功能 - 竞品调研报告

## 一、调研目标

为 FreeVideo 万能视频下载器新增 **AI 视频总结**功能，通过调研 BibiGPT 和 NoteGPT 两款竞品，了解市场现状、核心功能、交互设计和商业模式，为方案设计提供参考。

## 二、竞品概览

| 维度 | BibiGPT | NoteGPT |
|------|---------|---------|
| 官网 | bibigpt.co | notegpt.io |
| 定位 | AI 音视频知行助理 | AI 全能学习助手 |
| 用户量 | 100 万+ | 8000 万+ Learners |
| 支持平台 | 30+ (B站、YouTube、播客、小红书、TikTok等) | 主打 YouTube + B站 |
| 核心功能 | 总结、思维导图、字幕、问答、视觉化分析、AI改写 | 总结、思维导图、字幕、问答、多语言翻译 |
| 客户端 | Web + 浏览器插件 + 桌面客户端 + 微信助理 | Web + 浏览器插件 |

### 3.2 核心功能详解

#### 总结结果页（BibiGPT 为例，最完整）

BibiGPT 采用 **5 个 Tab 标签页** 组织总结内容：

| Tab | 内容 | 说明 |
|-----|------|------|
| 全文总结 | 整体摘要 + 章节摘要 | 默认展示，结构化总结 |
| 原文细读 | 逐段详细内容 + 时间戳 | 深度阅读模式 |
| 高光笔记 | 用户标记的重点 | 可二次编辑 |
| 字幕脚本 | 完整字幕 + 时间轴 | 点击跳转对应位置 |
| 视觉化总结(Beta) | 思维导图 + 图表 | 内容可视化呈现 |

## 四、交互设计对比

### 4.1 输入方式

| 维度 | BibiGPT | NoteGPT |
|------|---------|---------|
| 输入框位置 | 独立页面居中 | 首页 Hero 区居中 |
| 输入方式 | 4种：链接/探索/上传/网盘 | 1种：链接 |
| 批量输入 | ✅ Shift+Enter 换行 | ❌ |
| 示例链接 | 多平台示例 | B站示例 |
| 快捷操作 | 快速粘贴、总结设置、模型选择 | 仅 Generate 按钮 |

### 4.2 使用流程

**BibiGPT（6步）**：
1. 访问首页 → 点击"立即免费体验"
2. 选择输入方式（链接/上传/网盘）
3. 粘贴链接 → 点击"一键总结"
4. 等待 AI 处理
5. 查看结果 → 切换 Tab 浏览不同维度
6. 操作内容 → 复制/导出/AI对话

**NoteGPT（3步）**：
1. 首页输入链接 → 点击 Generate Summary
2. 等待处理（5秒~3分钟）
3. 获取总结 + 思维导图 + 转录文本

### 4.3 等待体验

| 维度 | BibiGPT | NoteGPT |
|------|---------|---------|
| 加载时间 | 未明确 | 5秒（有字幕）/ 2-3分钟（无字幕） |
| 加载反馈 | 加载动画 | 插图 + 进度条 + 趣味文案 |
| 用户体验 | 标准 | 较好，缓解等待焦虑 |

### 5.2 商业模式亮点

**BibiGPT**：
- 7天退款，决策压力低
- 合伙人制度（$888 永久买断 + 33%返现），适合 KOL 推广
- 微信生态深度整合

**NoteGPT**：
- 年付不自动续订，用户友好
- 明确的配额体系（Basic Quota + Premium Credits）
- 教育套餐专项优惠

## 七、对 FreeVideo 的启示

### 7.1 我们的差异化优势

FreeVideo 已有 **视频下载** 功能，新增 AI 总结可以形成 **"下载 + 理解"一站式闭环**，这是竞品没有的：

```
竞品：用户看到链接 → 去竞品总结 → 想下载再找工具
我们：用户粘贴链接 → 解析视频 → 选择"AI总结"或"下载" → 一站搞定
```

### 7.2 MVP 功能建议

基于竞品调研，AI 总结 MVP 建议实现以下核心功能：

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 一键总结 | 整体摘要 + 章节摘要（带时间戳） |
| P0 | 字幕提取 | 完整转录文本，带时间戳 |
| P1 | 思维导图 | 总结内容的可视化呈现 |
| P1 | AI 问答 | 基于视频内容的对话式提问 |
| P2 | 多语言翻译 | 总结/字幕的多语言切换 |
| P2 | 导出 | 支持 Markdown/PDF 导出 |

### 7.3 交互设计建议

1. **复用现有输入框**：不新建页面，在当前解析结果页新增"AI 总结"按钮
2. **Tab 标签页展示结果**：参考 BibiGPT 的 Tab 设计（总结 / 思维导图 / 字幕）
3. **简洁优先**：参考 NoteGPT 的简洁风格，避免过度复杂
4. **免费体验**：提供每日免费次数，降低用户尝试门槛

### 7.4 技术方案初步思考

| 模块 | 方案 |
|------|------|
| 字幕获取 | yt-dlp 提取字幕 / Whisper 语音转文字 |
| AI 总结 | 调用大模型 API（GPT/Claude/DeepSeek） |
| 思维导图 | 前端 Markmap 渲染 |
| AI 问答 | 基于字幕上下文的对话式 AI |
