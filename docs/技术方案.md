## 三、API 接口设计

### 3.1 解析视频信息

```
POST /api/parse
```

**请求体：**
```json
{
  "url": "https://www.youtube.com/watch?v=xxxxx"
}
```

**响应体：**
```json
{
  "success": true,
  "data": {
    "title": "视频标题",
    "thumbnail": "https://xxx/cover.jpg",
    "duration": 360,
    "uploader": "作者名",
    "formats": [
      {
        "format_id": "137+140",
        "ext": "mp4",
        "resolution": "1920x1080",
        "quality": "1080p",
        "filesize": 52428800,
        "has_video": true,
        "has_audio": true
      }
    ]
  }
}
```

### 3.2 开始下载

```
POST /api/download
```

**请求体：**
```json
{
  "url": "https://www.youtube.com/watch?v=xxxxx",
  "format_id": "137+140"
}
```

**响应体：**
```json
{
  "success": true,
  "task_id": "uuid-xxxx-xxxx"
}
```

### 3.3 下载进度（SSE）

```
GET /api/progress/{task_id}
```

**SSE 事件流：**
```
data: {"status": "downloading", "percentage": 45.2, "speed": "2.5MB/s", "eta": 30}
data: {"status": "downloading", "percentage": 89.1, "speed": "3.1MB/s", "eta": 5}
data: {"status": "finished", "filename": "video.mp4"}
```

### 3.4 下载文件

```
GET /api/file/{task_id}
```

返回文件流，浏览器触发下载。

### 3.5 清理文件

```
DELETE /api/file/{task_id}
```

## 四、核心实现方案

### 4.1 yt-dlp 集成（Python 库方式）

直接 `import yt_dlp` 作为 Python 库使用，而非命令行调用：

```python
from yt_dlp import YoutubeDL

# 解析视频信息（不下载）
def parse_video(url: str):
    ydl_opts = {'quiet': True, 'no_warnings': True}
    with YoutubeDL(ydl_opts) as ydl:
        info = ydl.extract_info(url, download=False)
        return ydl.sanitize_info(info)

# 下载视频（带进度回调）
def download_video(url: str, format_id: str, progress_callback):
    ydl_opts = {
        'format': format_id,
        'outtmpl': 'downloads/%(title)s.%(ext)s',
        'progress_hooks': [progress_callback],
        'merge_output_format': 'mp4',
    }
    with YoutubeDL(ydl_opts) as ydl:
        ydl.download([url])
```

### 4.2 下载进度推送（SSE）

使用 Server-Sent Events 实现实时进度推送：


### 4.3 临时文件管理

- 下载完成的文件存放在 `backend/downloads/` 目录
- 用户下载文件后 / 超过 30 分钟自动清理
- 使用 `task_id`（UUID）映射文件路径，防止路径遍历攻击

## 七、部署方案

### 开发环境
- 前端：`npm run dev`（Vite 开发服务器，默认 5173 端口）
- 后端：`uvicorn main:app --reload`（默认 8000 端口）
- Vite 配置代理，开发时前端请求自动转发到后端

### 生产环境
- 前端：`npm run build` 生成静态文件，Nginx 托管
- 后端：Uvicorn + Gunicorn 部署，或 Docker 容器化
- Nginx 反向代理，前后端统一入口

## 八、风险与应对

| 风险 | 应对措施 |
|------|---------|
| yt-dlp 被平台封禁 | 定期更新 yt-dlp 版本，使用代理 |
| 服务器磁盘空间不足 | 定时清理临时文件，限制单文件大小 |
| 下载滥用 | IP 频率限制，免费用户限制次数 |
| 版权法律风险 | 显著免责声明，仅供学习用途 |
| FFmpeg 未安装 | 启动时检查依赖，给出安装提示 |
