# 万能视频下载网站 - 第一期开发总结

## 一、项目概述

**FreeVideo** 是一个在线万能视频下载工具，支持 YouTube、Bilibili、TikTok、抖音、Instagram 等 **1700+** 平台的视频解析与下载。用户只需粘贴视频链接，即可解析视频信息、选择清晰度并一键下载。

- **前端**：Vue 3 + TypeScript + Vite + Tailwind CSS 4
- **后端**：Python FastAPI + yt-dlp + 抖音专用解析模块
- **通信**：RESTful API + SSE (Server-Sent Events) 实时进度推送

## 三、关键技术决策与解决方案

### 3.1 抖音专用解析模块

**问题**：yt-dlp 解析抖音视频需要浏览器 Cookie 验证（反爬虫机制），且不支持 `/jingxuan?modal_id=xxx` 等非标准 URL 格式。

**方案**：为抖音单独实现 `services/douyin.py` 解析模块：

- **URL 规范化**：支持 `modal_id` 参数、`/video/xxx` 路径、`v.douyin.com` 短链接等多种格式
- **双重解析**：先尝试 `iesdouyin.com` 公开 API，失败后 fallback 到分享页 HTML 中的 `_ROUTER_DATA` 解析
- **WAF 挑战破解**：内置 SHA-256 验证挑战解决器
- **无水印下载**：将播放地址中的 `playwm` 替换为 `play` 获取无水印视频
- **路由分发**：`is_douyin_url()` 自动识别抖音链接，走专用通道；其他平台继续走 yt-dlp

参考开源实现：[rathodpratham-dev/douyin_video_downloader](https://github.com/rathodpratham-dev/douyin_video_downloader)（MIT License）

### 3.2 Bilibili 缩略图防盗链处理

**问题**：Bilibili 图片 CDN 通过 `Referer` 头防盗链，外部域名请求图片被 403 拒绝。

**方案**：在 `<img>` 标签添加 `referrerpolicy="no-referrer"`，让浏览器不发送 Referer 头。

### 3.3 视频格式智能合并

**问题**：许多平台的高清视频流是音视频分离的（视频流 + 音频流），用户不理解 `format_id` 的组合。

**方案**：`_build_combined_formats()` 自动将最佳音频流与各清晰度视频流组合，生成用户友好的格式选项（如 "1080p (mp4)"），后端使用 FFmpeg 自动合并。

### 3.4 错误提示可见性

**问题**：解析失败时错误提示在 `VideoResult` 组件内部，该组件受 `v-if="videoInfo"` 控制，解析失败时不渲染，用户看不到错误。

**方案**：在 `Home.vue` 中 `VideoResult` 之前增加独立的错误提示区域，确保解析失败时错误信息对用户可见。
